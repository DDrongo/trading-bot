### Правила работы:
1. **Каждый дочерний чат получает 3-5 файлов** для работы
2. **Чёткое ТЗ** что нужно сделать
3. **Результат работы** — отчёт и изменённые файлы
4. **Отчёт возвращается в главный чат**

## 🎯 РАЗДЕЛ 2: ЦЕЛЕВОЕ СОСТОЯНИЕ

### 2.1 Три типа сигналов (нужно добавить)

| Тип | Таймфрейм | R/R | Время жизни | Статус |
|-----|-----------|-----|-------------|--------|
| **WATCH** | H4 | — | — | ❌ отсутствует |
| **LIMIT** | H4 | ≥ 3:1 | 24 часа | ✅ есть |
| **INSTANT** | M15 | ≥ 2:1 | 3 часа | ❌ отсутствует |

### 2.2 Полный цикл сделки (цель)
```
1. Генерация сигнала → событие TRADING_SIGNAL_GENERATED
2. Position Manager получает событие → открывает позицию
3. Мониторинг цены → проверка TP/SL
4. Закрытие → событие TRADE_CLOSED
5. Расчёт PnL → сохранение в историю
```

## 📌 ВАЖНО: ПРОЦЕСС РАБОТЫ

1. **Менеджер (этот чат)** создаёт ТЗ для дочернего чата
2. **В ТЗ обязательно указывает:**
   - Какие файлы нужно изменить (3-5 файлов)
   - Конкретную задачу
   - Требования к результату
3. **Дочерний чат** выполняет работу и возвращает:
   - Изменённые файлы
   - Отчёт о проделанной работе
   - Тесты (если требуются)
4. **Менеджер** проверяет, принимает работу, обновляет документацию
