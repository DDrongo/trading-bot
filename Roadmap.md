- LIMIT: R/R ≥ 3:1, время жизни 24ч
- Миграция БД (поля `signal_subtype`, `expiration_time`)
- Тесты: 12 тестов, все пройдены
- Обновлён `config.yaml` (секция `signal_types`)

### ФАЗА 1.3.1: Обновление монитора и исправление логики ордеров ⏳
**Причина:** После ночного тестирования выявлены критические недостатки, требующие немедленного исправления.

**Задачи:**
- [ ] Position Manager: различать LIMIT и INSTANT
  - LIMIT → лимитный ордер → статус `PENDING` → ожидание цены → `ACTIVE`
  - INSTANT → рыночный ордер → `ACTIVE` (сразу)
- [ ] Мониторинг лимитных ордеров (достижение цены)
- [ ] Добавить поле `screen` в БД (D1/H4/M15)
- [ ] Добавить поле `order_type` в БД (LIMIT/MARKET)
- [ ] Добавить поле `fill_price` в БД (цена исполнения)
- [ ] Добавить поле `position_size` в БД (средства в сделке)
- [ ] Монитор: таблица сигналов
  - Добавить колонки: Экран, Тренд, Средства
  - Убрать "Силу сигнала" и "Силу тренда"
  - Преобразование UTC → локальное время
- [ ] Монитор: таблица сделок (пункт меню)
  - Отображение `trades` с колонками: ID, Монета, Тип, Entry, Close, PnL, PnL %, Причина, Время
  - Статистика: win rate, total PnL, avg PnL
- [ ] Монитор: детали сигнала по ID
  - Полная переработка: блоки D1, H4, M15, параметры, текущая цена, расстояния
  - Дружелюбные пояснения на русском
- [ ] Исправить дублирование сигналов
- [ ] Исправить ADAUSDT (цены 0.28)

**Файлы для работы:**
